情況 C — 新增 index.html 首頁檔案
GitHub Pages 要求首頁必須命名為 index.html，檔名錯誤就會 404
❌
根本原因：GitHub Pages 的首頁檔案必須命名為 index.html。如果 Repo 裡只有 README.md 或 admin.html 等其他名稱，就會顯示 404。
⏱️
預估時間：5 分鐘（上傳檔案 + 等待部署）
1
Step 01

確認 Repo 裡的檔案名稱

前往 github.com/atomy-ai/cps，看 Code 分頁裡的檔案列表

github.com/atomy-ai/cps
📄admin.html← ❌ 不是 index.html！
📄README.mdInitial commit
⚠️如果看到的是 admin.html 或其他名稱，需要改名或另外建一個 index.html
2
Step 02

選擇方式建立 index.html

方式一：直接在 GitHub 新增 推薦
1. 在 Repo 頁面點 Add file → Create new file
2. 檔案名稱填 index.html
3. 內容先填 <h1>Hello</h1> 測試
4. 點 Commit changes
方式二：上傳本地 HTML 檔案
1. 把要放的 HTML 改名為 index.html
2. 在 Repo 頁面點 Add file → Upload files
3. 拖入 index.html
4. 點 Commit changes
3
Step 03

上傳後 GitHub 會自動重新部署，通常 1～3 分鐘內完成。可到 Repo 的 Actions 分頁查看進度。

設定完 Pages 後，要等多久才能看到網站？

▼
⏱️
通常 1～3 分鐘。如果超過 5 分鐘還沒好，去 Actions 分頁看看有沒有紅色錯誤。第一次設定可能需要最多 10 分鐘。
🔴

我用的是 Private Repo，為什麼 Pages 開不起來？

▼
💳
Private Repo 要用 GitHub Pages 需要付費升級到 Pro 方案（每月約 $4 美元）。
最簡單的解法：到 Repo Settings → General → 最底下「Danger Zone」→ Change visibility → 改為 Public（公開）即可免費使用 Pages。
🌐

我的網址為什麼是 atomy-ai.github.io/cps/ 而不是 atomy-ai.github.io/？

▼

因為你的 Repo 名稱是 cps，所以網址有 /cps/ 路徑。

如果想要根網址 atomy-ai.github.io/，需要把 Repo 改名為 atomy-ai.github.io（格式：帳號名.github.io），這種 Repo 叫做「User Pages」。

💡
對 CPS 系統來說，/cps/ 路徑就很夠用了，不需要特別改。
📝

我更新了 index.html，為什麼網站還是舊的？

▼

有兩個可能：

1. 瀏覽器快取： 按 Ctrl + Shift + R（Windows）或 Cmd + Shift + R（Mac）強制重新整理。

2. 部署還沒完成： 去 Actions 分頁確認最新的部署是否已是綠色 ✅。

💡

我可以放多個 HTML 頁面嗎？連結怎麼寫？

▼

可以！把所有 HTML 都上傳到 Repo 根目錄，連結用相對路徑。

<!-- ✅ 正確：相對路徑 --> <a href="deploy-guide.html">部署指南</a> <a href="shopee-affiliate.html">蝦皮教學</a> <!-- ❌ 錯誤：不要用絕對路徑 --> <a href="/deploy-guide.html">錯誤示範</a>
🔗

我想用自己的域名（例如 deals.mybrand.com），怎麼設定？

▼

需要兩個步驟：①購買域名 ②設定 CNAME

1. 在你的域名商後台，新增一筆 CNAME 記錄：名稱填 deals，值填 atomy-ai.github.io

2. 到 GitHub Repo → Settings → Pages → Custom domain，輸入 deals.mybrand.com 後按 Save

3. 等待 5～30 分鐘 DNS 生效，GitHub 會自動申請 HTTPS 憑證

💡
推薦到 Namecheap 購買 .com 域名（約 NT$350/年），在 CPS 系統初期不是必要的，有了穩定流量再買也不遲。
