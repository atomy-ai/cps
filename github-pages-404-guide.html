你看到這個錯誤，代表 GitHub Pages 尚未正確設定。請選擇下方符合你狀況的情境，依步驟操作。

情況 A 解決方案
建立新的 GitHub Repo
1
Step 01

登入 GitHub 並建立新 Repo

確認使用 atomy-ai 帳號登入，點右上角的 ＋ → New repository

github.com/new

✅ Add a README file
👆Repository name 填 cps，選 Public（私有 repo 需要付費才能開 Pages），勾選 README，然後按 Create
2
Step 02

上傳你的 index.html 檔案

建立 repo 後，點 Add file → Upload files，把你的 HTML 檔案拖進去，確認檔名是 index.html

⚠️首頁檔案一定要叫 index.html，不能是 admin.html 或其他名稱，否則會顯示 404
3
Step 03

開啟 GitHub Pages

接著按照下方「情況 B」的步驟開啟 Pages 功能即可。

情況 C 解決方案
新增 index.html 首頁
❌GitHub Pages 的首頁檔案必須命名為 index.html。如果 repo 裡只有 README.md 或其他名稱的 HTML，就會顯示 404。
1
Step 01

確認 repo 裡的檔案

前往 github.com/atomy-ai/cps，看看 Code 分頁裡有什麼檔案

github.com/atomy-ai/cps
📁
.
Initial commit
📄
admin.html
← ❌ 不是 index.html！
📄
README.md
Initial commit
⚠️如果首頁是 admin.html 或其他名稱，需要改名或另外建一個 index.html
2
Step 02

選擇其中一種方式建立 index.html

方式一：直接在 GitHub 上新增（最簡單）
1. 在 repo 頁面點 Add file → Create new file
2. 檔案名稱填 index.html
3. 內容可以先填一行 <h1>Hello</h1> 測試
4. 點 Commit changes
方式二：上傳本地的 HTML 檔案
1. 把你要放的 HTML 檔案重新命名為 index.html
2. 在 repo 頁面點 Add file → Upload files
3. 把 index.html 拖進上傳區域
4. 點 Commit changes
3
Step 03

等待 GitHub Actions 部署完成

上傳後 GitHub 會自動重新部署，通常 1～3 分鐘內完成。可以到 repo 的 Actions 分頁查看進度。

github.com/atomy-ai/cps/actions
✅
pages build and deployment
completed 1 minute ago
⏳
上一次部署（進行中...）
✅看到綠色 ✅ 就代表部署完成，去 https://atomy-ai.github.io/cps/ 重新整理即可
🎉

設定完成後你的網站網址

完成上述步驟並等待部署後，用瀏覽器打開：

https://atomy-ai.github.io/cps/

若還是 404，請清除瀏覽器快取（Ctrl+Shift+R）再試一次
